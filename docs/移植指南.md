## 完整移植步骤

### 1. 需要复制的目录结构

```
sample_import/                    # 主目录（可重命名）
├── board_controller/             # ✅ 必需 - 单板控制模块
├── mcp_server/                   # ✅ 必需 - MCP 工具服务器
├── core_utils/                   # ✅ 必需 - 核心工具
├── skills/                       # ✅ 必需 - 分析技能（11个）
├── config/                       # ✅ 必需 - 配置文件
│   └── board_config.yaml
├── tests/                        # 可选 - 测试文件
├── docs/                         # 可选 - 文档
├── .claude/                      # ⚠️ 注意 - Claude Code 配置
│   └── settings.local.json
├── requirements.txt              # ✅ 必需 - Python 依赖
└── README.md                     # 可选 - 说明文档
```

### 2. 可省略的目录/文件

```
以下内容不需要复制：
├── .git/                         # Git 仓库
├── .pytest_cache/                # 测试缓存
├── __pycache__/                  # Python 缓存
├── state/                        # 状态文件（会在目标机器生成）
├── logs/                         # 日志文件
└── skills.zip                    # 压缩包（已有 skills/ 目录）
```

### 3. 移植步骤

#### 步骤 1: 复制项目

```bash
# 方式一：打包复制（推荐）
# 在源机器上
cd D:\ai_dir\
tar -czf sample_import.tar.gz sample_import/

# 复制到目标机器，然后解压
tar -xzf sample_import.tar.gz

# 方式二：直接复制目录
# 直接复制整个 sample_import 文件夹到目标机器
```

#### 步骤 2: 安装依赖

```bash
# 进入项目目录
cd sample_import

# 安装 Python 依赖
pip install -r requirements.txt

# 如果需要真实硬件测试，额外安装：
pip install pyserial asyncssh
```

#### 步骤 3: 验证安装

```bash
# 运行测试
pytest tests/ -v

# 应该看到 34 tests passed ✅
```

#### 步骤 4: 配置调整（可选）

##### 4.1 更新单板配置

如果硬件配置不同，修改 `config/board_config.yaml`：

```yaml
local_board:
  mode: local
  serial:
    port: COM3           # 根据实际情况修改
    baudrate: 115200
  ping:
    host: 192.168.1.100  # 根据实际情况修改
```

##### 4.2 更新 Claude Code 配置

`.claude/settings.local.json` 中的路径配置可能需要调整。

## 注意事项

### ⚠️ 重要提醒

| 项目 | 说明 |
|------|------|
| **Python 版本** | 需要 Python 3.8 或更高 |
| **工作目录** | 确保在项目目录或其父目录中使用 Claude Code |
| **网络访问** | 如果使用 WebSearch 技能，需要网络连接 |
| **硬件依赖** | 如果不进行真实硬件测试，不需要 pyserial/asyncssh |

### 🔧 可能需要的调整

1. **串口配置**：Linux 是 `/dev/ttyUSB0`，Windows 是 `COM3`
2. **路径配置**：如果项目目录名改变，需要更新相关配置
3. **权限配置**：根据需要调整 `.claude/settings.local.json`

## 最小安装（仅核心功能）

如果只需要核心分析功能（不使用真实硬件），最小需要：

```
sample_import/
├── skills/                   # 必需
├── core_utils/              # 必需
├── mcp_server/              # 必需（Mock 模式）
├── requirements.txt         # 必需
└── README.md                # 建议
```

可以省略：
- `board_controller/`（不需要硬件控制）
- `config/`（不需要硬件配置）
- `tests/`（不需要测试）

## 快速移植命令

### Windows

```powershell
# 打包
Compress-Archive -Path D:\ai_dir\sample_import -DestinationPath sample_import.zip

# 解压到目标机器
Expand-Archive -Path sample_import.zip -DestinationPath D:\target_path\
```

### Linux/Mac

```bash
# 打包
tar -czf sample_import.tar.gz sample_import/

# 解压
tar -xzf sample_import.tar.gz -C /target/path/
```

## 故障排查

| 问题 | 解决方案 |
|------|----------|
| 导入错误 | `pip install -r requirements.txt` |
| 测试失败 | 检查 Python 版本 >= 3.8 |
| Skills 未识别 | 确认在项目目录中使用 Claude Code |
| 硬件连接失败 | 检查配置文件，或使用 Mock 模式 |
