# General Diagnose Skill

## 描述

对芯片日志进行通用诊断分析，当无法明确归因为时钟、电源或温度问题时使用。涵盖逻辑错误、协议违规、配置错误等通用问题。

## 触发条件

- 日志中存在错误/异常但不符合专业领域特征
- 需要综合多个领域信息进行分析
- 作为初始分析的默认入口

## 分析步骤

1. **扫描错误模式**
   - 识别 ERROR, FAIL, WARNING 等关键字
   - 统计错误频率和分布
   - 识别错误发生的时间模式

2. **分类问题类型**
   - 硬件故障：寄存器错误、总线错误
   - 软件错误：异常、断言失败
   - 配置错误：参数设置不当
   - 通信错误：协议违规、超时

3. **分析上下文**
   - 查看错误前后的日志
   - 识别触发条件
   - 分析错误传播路径

4. **生成假设**

   假设格式：
   ```json
   {
     "id": "H4",
     "category": "general",
     "content": "AXI 总线写响应超时",
     "confidence": 0.7,
     "evidence": ["log_line_345", "log_line_678"],
     "test_history": []
   }
   ```

## 输出

返回分析结果，包含：
- 发现的问题列表
- 每个问题的置信度
- 建议的测试方案
- 相关日志行号
- 如果适合，建议转给专业分析 Skill

## 常见问题模式

1. **总线错误**
   - 症状：事务超时，响应错误
   - 证据：`timeout`, `error response`, `bus fault`
   - 测试：使用 `general_test` 工具

2. **中断异常**
   - 症状：中断丢失，风暴，嵌套错误
   - 证据：`interrupt`, `IRQ`, `handler`
   - 测试：使用 `general_test` 工具

3. **DMA 错误**
   - 症状：传输失败，数据损坏
   - 证据：`DMA error`, `transfer fail`
   - 测试：使用 `general_test` 工具

4. **寄存器错误**
   - 症状：读写失败，值异常
   - 证据：`register access`, `RW error`
   - 测试：使用 `general_test` 工具

5. **状态机异常**
   - 症状：状态卡死，跳转错误
   - 证据：`state machine`, `stuck`, `illegal state`
   - 测试：使用 `general_test` 工具

6. **复位问题**
   - 症状：意外复位，复位失败
   - 证据：`reset`, `POR`, `watchdog`
   - 测试：使用 `general_test` 工具

## 分析建议

- 优先关注首次出现的错误
- 注意错误之间的时间关联
- 考虑工作负载和操作序列的影响
- 检查是否有资源耗尽（内存、缓冲区等）
