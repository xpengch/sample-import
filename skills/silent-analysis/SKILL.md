# Silent Failure Analysis Skill

## 描述

分析芯片日志中的静默失败问题，即没有明显错误日志但系统行为异常的问题。

## 触发条件

当以下情况时触发：
- 用户报告功能异常但日志无明显错误
- 数据结果不正确但无错误记录
- 性能下降但无报错
- 状态不一致但无异常日志
- 需要通过对比分析才能发现问题

## 分析步骤

1. **识别异常行为**
   - 收集用户报告的症状
   - 对比正常和异常时的日志
   - 识别行为差异点

2. **检测静默失败特征**
   - 搜索返回值异常（无错误处理）
   - 查找状态不一致证据
   - 识别时序异常

3. **分析根因**
   - 检查错误处理缺失
   - 验证状态机逻辑
   - 分析数据完整性

4. **生成假设**

   假设格式：
   ```json
   {
     "id": "H30",
     "category": "silent",
     "content": "寄存器写入失败但��检测",
     "confidence": 0.7,
     "evidence": ["log_line_xxx", "log_line_yyy"],
     "test_history": []
   }
   ```

## 输出

返回分析结果，包含：
- 发现的静默失败问题列表
- 每个问题的置信度
- 建议的测试方案
- 相关日志行号

## 常见问题模式

1. **写操作失败未检测**
   - 症状：配置未生效
   - 证据：写后读不一致
   - 测试：待补充

2. **状态机不一致**
   - 症状：行为与预期不符
   - 证据：状态跳转异常
   - 测试：待补充

3. **数据静默损坏**
   - 症状：计算结果错误
   - 证据：数据校验失败
   - 测试：待补充

4. **初始化静默失败**
   - 症状：模块功能异常
   - 证据：默认值未改变
   - 测试：待补充

5. **中断静默丢失**
   - 症状：事件未响应
   - 证据：中断计数不匹配
   - 测试：待补充

6. **缓存一致性问题**
   - 症状：内存数据陈旧
   - 证据：缓存与内存不一致
   - 测试：待补充
